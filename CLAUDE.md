## Commands

```bash
# Install dependencies
bundle install

# Run local dev server (http://localhost:4000)
bundle exec jekyll serve

# Run with drafts visible
bundle exec jekyll serve --drafts

# Build static site to _site/
bundle exec jekyll build
```

## Architecture

This is a Jekyll static blog hosted on GitHub Pages. Pushing to `main` triggers automatic deployment.

**Layouts** (`_layouts/`) — Liquid template hierarchy:
- `default.html` — base shell: topbar, sidebar, main content area, footer, JS includes
- `home.html` — extends default; renders hero, featured posts, category chips, paginated post list
- `post.html` — extends default; renders article with breadcrumbs, metadata, related posts, Giscus comments
- `page.html` — extends default; generic static page wrapper

**Pages** (`pages/`) — Static sections with custom layouts inside `_layouts/`: `categorias.html`, `archivo.html`, `anecdotario.html`, `contact.html`, `descargas.md`.

**Includes** (`_includes/`) — Reusable Liquid components: `sidebar.html`, `head.html`, `post-card.html`, `related-posts.html`, `comments.html`, `breadcrumbs.html`, `footer.html`.

**Styles** (`_sass/` + `assets/css/main.scss`) — Custom SCSS, no external CSS framework. Entry point is `assets/css/main.scss` which imports all partials. Theme switching (dark/light) is implemented via CSS custom properties on `data-theme` attribute; `_variables.scss` defines both themes and all shared tokens.

**JS** (`assets/js/`) — Vanilla JS only:
- `theme.js` — persists and applies dark/light theme preference
- `search.js` — client-side search against `search.json` (generated by Jekyll from post metadata)

**Comments** — Giscus (GitHub Discussions–backed). Config lives in `_config.yml` under the `giscus:` key.

## Writing Posts

Posts go in `_posts/` with the filename format `YYYY-MM-DD-slug.md`. Required frontmatter:

```yaml
---
title: Título del post
description: Descripción breve
date: 2024-01-15 10:00:00 +0100
categories: [Java]
tags: [java, collections, streams]
---
```

To mark a post as featured (shown in the home hero grid), add `featured: true` to the frontmatter.

Categories drive the `/categorias/` page and the sidebar navigation chips. Tags appear at the bottom of each post and also link to `/categorias/`.

## Key Config

- `_config.yml` — site metadata, permalink structure (`/posts/:year/:month/:day/:title/`), pagination (10 per page), SCSS compression, Giscus settings
- `CNAME` — custom domain binding for GitHub Pages
- `search.json` — Liquid template that generates the JSON index consumed by `search.js`
